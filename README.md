# Quals Presentation

Quarto RevealJS presentation (`clean-revealjs` theme). See `CLAUDE.md` for project structure and styling details.

## View the presentation (HTML)

Render once, then open `main.html` in a browser:

```bash
quarto render main.qmd
xdg-open main.html   # or: open main.html (macOS)
```

Or use live preview (auto-reloads on edits):

```bash
quarto preview main.qmd
```

Some plots are generated by scripts in `scripts/` using the `reward-exploration` conda environment.

## Generate a PDF

The PDF is produced by [decktape](https://github.com/astefanutti/decktape), which drives a headless browser through the slides so styling and fragment (step-by-step reveal) behavior match the HTML version exactly. Quarto's built-in PDF export does *not* preserve RevealJS styling or fragments, which is why we use decktape.

### One-time setup

Install Node (via [nvm](https://github.com/nvm-sh/nvm)) and decktape:

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
export NVM_DIR="$HOME/.nvm" && . "$NVM_DIR/nvm.sh"
nvm install --lts
npm install -g decktape
```

### Build the PDF

```bash
quarto render main.qmd
decktape reveal --fragments "file://$(pwd)/main.html" main.pdf --size 1280x720
```

Output: `main.pdf` in the project root. The `--fragments` flag makes each fragment step its own page (without it, decktape only captures the final state of each slide).

### Troubleshooting

- **`decktape: command not found`** — run `nvm use --lts` in a new shell, or source nvm: `. "$HOME/.nvm/nvm.sh"`.
- **Stale content in PDF** — re-run `quarto render main.qmd` before decktape; the PDF is built from `main.html`.
- **Different page size** — change `--size 1280x720` (width x height in pixels).
